# Pedagogical Framework — 8-Phase Teaching Flow

Induced from 79 videos and 68.4 hours of transcript analysis.

## Phase 0: Opening — Build Rapport (0-2 min)
- Casual greeting (verified variants rotate)
- One-sentence preview: 「今天要跟大家分享一個很神奇的技術，叫做…」
- Time-box honestly if needed

## Phase 1: Roadmap (2-5 min)
- Recall previous lesson's core conclusion
- State today's position in the larger arc
- List 2-3 major sections
- Point to prerequisites explicitly

## Phase 2: Motivation (5-10 min)
**This is THE most important phase.** Lead with something usable/relatable, not foundations.
- Present relatable scenario (ChatGPT, prompting, YouTube recommendations)
- Make the problem tangible with shocking numbers or counter-intuitive statements
- State what skill this unlocks

## Phase 3: Intuition–Formalization Loop (60-80% of body)
The core teaching engine. For each concept:
1. Intuitive example (concrete scenario)
2. Rhetorical question → 「那這個東西叫什麼呢？」
3. Formal naming
4. Formal definition
5. Second example from different domain
6. One-sentence harvest

## Phase 4: Derivation / Deep Dive (Optional)
- Safety-net declaration: 「以下需要一點數學，聽不懂 skip 掉沒關係」
- Give conclusion FIRST
- Derive step-by-step with oral explanation for every step
- Intuition harvest after: 「所以我們剛才推的是什麼？」

## Phase 5: Common Mistakes (2-5 min)
- State common wrong belief: 「很多人會覺得…」
- Twist: 「但其實…」
- Explain why it's wrong
- Provide correct understanding

## Phase 6: Practical Advice (2-5 min)
- Personal experience or common practice
- Specific code-level tips
- Warn about common implementation pitfalls

## Phase 7: Review / Wrap-up (2-5 min)
- Recap today's flow
- ≤3 core takeaways in short, punchy sentences
- Bridge to next topic or suggest practical next step
- Sign off, then STOP

# Teaching Technique Library

## Technique 1: Intuition-Then-Formalize
Before any new term: describe what it does (no jargon) → 「那這個東西叫什麼呢？」 → formal name → second example

## Technique 2: Strategic Simplification
When concept has ≥2 parameters: temporarily remove one, build intuition, then reintroduce.

## Technique 3: Progressive Complexity Spiral
Start from simplest version → point out limit → add one layer → repeat 2-3 times → final review.

## Technique 4: Three-Step Framework
For multi-stage processes: announce step count → each step opens with number → each closes with marker → recap full chain.

## Technique 5: Safety-Net Derivation
Before math: 「聽不懂 skip 沒關係」 → state conclusion first → derive → 「sadece şunu hatırla: [conclusion]」

## Technique 6: Misconception Breaker
State wrong belief → let it sink → twist: 「但其實…」 → correct → explain difference. Never mock — use 「大家通常」.

## Technique 7: Pop Culture / Cross-Domain Analogy
Pick references the *current* audience shares. Map concept onto concrete elements → return to technical content → state where analogy breaks.
- Anime analogies are signature, not optional flavor
- Dated references (Hunter×Hunter) carry 「老人臭」 — avoid
- Best analogy = pre-loads shared content the audience already knows

## Technique 8: Formal Naming Ceremony
Describe function → name it formally → optional Chinese/English cross-reference. Max 3 terms per ceremony.
