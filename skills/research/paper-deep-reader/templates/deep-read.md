## 🔬 模块 3：方法/技术内容

> 本模块是精读笔记的核心，需要全面、深入、形式丰富地解读论文的技术内容。
> 根据论文类型选择下方对应结构，但以下元素应尽可能覆盖：

### 3.1 问题形式化

将论文要解决的问题用数学/逻辑语言形式化描述：

- **输入**：系统的输入是什么？（传感器数据、状态变量、用户指令等）
- **输出**：期望的输出是什么？（控制指令、规划路径、分类结果等）
- **约束条件**：有哪些物理/数学/工程约束？
- **优化目标**：最小化/最大化什么指标？

> 形式化描述帮助读者精确理解问题边界，避免模糊理解。

### 3.2 方法概览

用**流程图或架构图**展示整体方法，配以简要文字说明：

```
输入 → 模块A → 模块B → 模块C → 输出
         ↑              ↓
       反馈环路 ← 模块D
```

- 每个模块的作用（一句话）
- 模块间的依赖关系
- 数据流向

### 3.3 核心算法/方法详解

**如有算法伪代码：**

| 步骤 | 操作 | 输入 | 输出 | 复杂度 |
|------|------|------|------|--------|
| 1 | ... | ... | ... | O(n) |
| 2 | ... | ... | ... | O(n²) |

**算法关键步骤解读：**
- 每步在做什么？为什么这样做？
- 有哪些条件判断/分支？
- 关键的数学依据是什么？

**如有核心公式：**

$$公式$$

- **符号解释**：逐个符号说明含义和单位
- **物理/数学意义**：这个公式在描述什么现象/关系？
- **在论文中的作用**：用于哪个步骤？解决什么子问题？
- **假设条件**：公式成立的前提条件是什么？
- **与同类方法的对比**：与其他论文的公式有什么区别？改进了什么？

### 3.4 关键设计决策

每个技术选择背后都有设计决策，需要分析：

| 设计决策 | 本文选择 | 替代方案 | 选择理由 | 权衡(trade-off) |
|----------|----------|----------|----------|-----------------|
| 模型架构 | ... | ... | ... | 精度vs速度 |
| 优化方法 | ... | ... | ... | 收敛速度vs稳定性 |
| 特征表示 | ... | ... | ... | 表达力vs计算量 |

> 分析"为什么这样选择"比"选择了什么"更重要。

### 3.5 与同类方法的对比分析

| 维度 | 本文方法 | 方法A | 方法B | 方法C |
|------|----------|-------|-------|-------|
| 核心思路 | ... | ... | ... | ... |
| 创新点 | ... | ... | ... | ... |
| 计算复杂度 | ... | ... | ... | ... |
| 适用场景 | ... | ... | ... | ... |
| 主要限制 | ... | ... | ... | ... |

> 这个对比表帮助读者快速理解本文方法在领域中的定位。

### 3.6 方法的优势与潜在问题

**优势：**
1. ...（与传统方法相比）
2. ...（与同类学习方法相比）

**潜在问题：**
1. ...（假设是否合理？）
2. ...（可扩展性如何？）
3. ...（实际部署的挑战？）

## 📊 模块 4：实验/结果/证据

> 本模块需要全面呈现实验设计、结果分析和结论验证。
> 不仅要展示"结果是什么"，更要分析"结果说明了什么"和"结果的可信度"。

### 4.1 实验方法论

#### 实验环境

| 项目 | 详情 |
|------|------|
| 硬件 | ...（机器人型号、传感器、计算平台等） |
| 软件 | ...（仿真环境、框架、版本等） |
| 数据集 | ...（来源、规模、划分方式） |
| 评估指标 | ...（指标名称、计算方式、为什么选这个指标） |

#### 实验设计

- **基线方法(Baseline)**：选择了哪些对比方法？为什么选这些？
- **实验变量**：自变量、因变量、控制变量分别是什么？
- **重复性**：每组实验重复几次？是否有随机种子控制？
- **统计检验**：是否使用了统计显著性检验？(t-test, Wilcoxon, etc.)

### 4.2 定量结果

#### 主要结果对比表

| 方法 | 指标1 | 指标2 | 指标3 | 指标4 |
|------|-------|-------|-------|-------|
| Baseline A | ... | ... | ... | ... |
| Baseline B | ... | ... | ... | ... |
| Baseline C | ... | ... | ... | ... |
| **本文方法** | **...** | **...** | **...** | **...** |
| 提升幅度 | +X% | +Y% | +Z% | +W% |

**结果解读：**
- 在哪些指标上显著优于 baseline？原因是什么？
- 在哪些指标上提升不明显？可能的原因？
- 是否有指标反而下降？如何解释？

#### 消融实验（如有）

| 配置 | 组件A | 组件B | 组件C | 指标 |
|------|-------|-------|-------|------|
| 完整模型 | ✓ | ✓ | ✓ | ... |
| 去掉A | ✗ | ✓ | ✓ | ... |
| 去掉B | ✓ | ✗ | ✓ | ... |
| 去掉C | ✓ | ✓ | ✗ | ... |

**消融分析：**
- 哪个组件贡献最大？
- 组件间是否有协同效应？
- 哪个组件可以去掉而不显著影响性能？

### 4.3 定性结果

**关键可视化解读：**

![Figure N: caption](images/filename.png)
> **观察**：...
> **与定量结果的关系**：...
> **值得注意的细节**：...

**典型成功案例：**
- 什么情况下方法效果最好？
- 成功的关键因素是什么？

**典型失败案例：**
- 什么情况下方法失败/效果差？
- 失败的根本原因是什么？
- 是否有改进方向？

### 4.4 计算开销分析

| 方法 | 参数量 | 推理时间 | 内存占用 | 训练时间 |
|------|--------|----------|----------|----------|
| Baseline A | ... | ... | ... | ... |
| **本文方法** | **...** | **...** | **...** | **...** |

- 是否满足实时性要求？
- 在不同硬件上的表现？
- 可扩展性如何？

### 4.5 结果的可信度评估

| 维度 | 评估 | 说明 |
|------|------|------|
| 实验规模 | 充分/一般/不足 | 数据集大小、场景多样性 |
| 基线选择 | 公正/偏弱/全面 | 是否包含SOTA方法 |
| 统计严谨性 | 有检验/无检验 | 是否报告置信区间/p值 |
| 可重复性 | 高/中/低 | 代码/数据是否公开 |
| 结果一致性 | 好/一般/差 | 不同场景下表现是否稳定 |

> 结果的可信度直接影响论文的说服力。

### 4.6 结果的局限性

- 实验条件与实际应用的差距
- 未测试的场景/条件
- 结果的泛化性讨论
